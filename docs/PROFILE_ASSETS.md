## Proposal style

Keep proposals short, specific, and scoped:

1. Confirm that the problem is understood.
2. Name the exact technologies involved.
3. State the first deliverable.
4. Ask only the minimum clarification needed.
5. Avoid promising production security/audits without reviewing the code first.

## Operational rule

For active ticket platforms, prioritize real, open, payable tasks. Small rewards
are allowed when they can build cash flow, reputation, portfolio proof, or
platform history. Do not start work when the payout path is unclear, the issue is
already assigned, a competing PR is active, the repository looks unsafe, or the
task requires upfront capital.
